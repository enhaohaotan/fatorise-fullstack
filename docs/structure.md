# API Contract — v1

## Base

Base URL: /api

Content-Type: application/json

Resource: Task
Fields
Field Type Required Notes
id string server UUID generated by server
title string yes trimmed, 1–100 chars
description string | null no 0–500 chars
completed boolean no default: false
createdAt string server ISO-8601 timestamp
updatedAt string server ISO-8601 timestamp
Validation Rules

title: required, trimmed, length 1–100

description: optional, length 0–500

completed: optional, boolean

Error Response (Unified)

All non-2xx responses MUST follow:

{
"error": {
"code": "VALIDATION_ERROR",
"message": "Invalid request body",
"details": [
{ "path": "title", "message": "Title is required" }
]
}
}

Error Codes

VALIDATION_ERROR → 400

NOT_FOUND → 404

INTERNAL_ERROR → 500

Endpoints

1. List Tasks

GET /tasks

200 OK

{
"items": [
{
"id": "uuid",
"title": "Buy milk",
"description": "2L whole milk",
"completed": false,
"createdAt": "2025-12-22T14:10:00.000Z",
"updatedAt": "2025-12-22T14:10:00.000Z"
}
]
}

2. Create Task

POST /tasks

Request body:

{
"title": "Buy milk",
"description": "2L whole milk",
"completed": false
}

201 Created (returns created Task)

{
"id": "uuid",
"title": "Buy milk",
"description": "2L whole milk",
"completed": false,
"createdAt": "2025-12-22T14:10:00.000Z",
"updatedAt": "2025-12-22T14:10:00.000Z"
}

400 Validation Error (example)

{
"error": {
"code": "VALIDATION_ERROR",
"message": "Invalid request body",
"details": [
{ "path": "title", "message": "Title is required" }
]
}
}

3. Update Task (Partial Update)

PATCH /tasks/{id}

Request body (any subset of fields):

{
"title": "Buy oat milk",
"completed": true
}

200 OK (returns updated Task)

404 Not Found

{
"error": {
"code": "NOT_FOUND",
"message": "Task not found",
"details": [{ "path": "id", "message": "No task exists for the provided id" }]
}
}

4. Delete Task

DELETE /tasks/{id}

204 No Content (no response body)

404 Not Found (same shape as above)

Notes

Server owns id, createdAt, updatedAt; client must not send these fields.

description may be omitted; server stores null (or empty string) consistently.

Any unexpected exception returns 500 INTERNAL_ERROR using the unified error shape.
